 What I'm working on

I'm working on building a set of standalone PHP libraries that can be easily integrated into any project, even a non-PHP one if you just want to use the javascript.
 
Links to my libraries:
  • BootstrapConstructor - provides functions for building Twitter Bootstrap HTML.
  • DatabaseManager - provides functions for database querying/management.
  • DataValidator - provides functions for client/server side validation.
  • LanguageHelper - provides multilanguage support functions.
  • DataFormatter - provides functions for formatter any kind of data.
  • JsonVC - provides a basic URL router library that sends back Json.
  • Logger - provides functions for writing to log files from both client and server.
  • SessionManager - parses HTTP headers to determine user platform / language info.

 End Goal

The end goal is to build an as yet unnamed project that uses all of these libraries together. This project will require at least jQuery and Twitter Bootstrap, and likely a few other misc. libraries such as lessphp.

 Coding style

All libraries share a common coding style in terms of class, function, and variable names. Each library has a 3 letter abbreviation: BootstrapConstructor's is 'bsc', JsonVC's is 'jvc'. This abbreviation is used to name the primary class in each framework and the single global variable used by each library. The global variable is named $__[abbreviation]. For example, $__jvc and $__bsc. This variable is always an array. This is to minimize the possibility of a naming collision with any porject.

 Testing

Each library comes with its own unit tests. They can be run from the command line using PHP, or via a web server. Each test generates an output file, which is then compared to an expected file.

 Including a library

To load any library, simply include the main PHP class file. Some libraries require an init function be called to pass configuration info. Some libraries may require a javascript file be loaded into your webpage to use client-side functions. Each library can be included as a git submodule, or svn external via github.
